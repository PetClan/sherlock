Theme Protection & Backups

Every day, Sherlock saves a complete copy of your theme ‚Äî every file, every line of code. We call this a theme snapshot.

Think of it like a time machine for your store's design. If an app breaks something, you can restore your theme to exactly how it looked on any previous day.

üí° What gets backed up

All theme files including: layouts, sections, snippets, templates, CSS, JavaScript, images, and configuration files.

Files backed up (e.g., 274) = The total number of unique files in your theme that Sherlock has captured. This includes everything: layouts, sections, snippets, CSS, JavaScript, images, etc.

Restore points (e.g., 2409) = The total number of file versions stored. Each daily scan saves a new version of each file.

Example: If you have 274 files and Sherlock has been running for 9 days, you'd have roughly 274 √ó 9 = 2,466 restore points.

Restoring Your Theme

When you click "Restore to [Date]", Sherlock:

  1. Looks up the saved version of each file from that date
  2. Sends each file back to Shopify one at a time
  3. Shopify overwrites the current file with the old version

You'll see real-time progress as each file is restored. A typical theme with ~274 files takes about 4-5 minutes.

This is a real restore ‚Äî your live store changes immediately. It's not a preview. Make sure you want to restore before clicking!

Daily Scans & Monitoring

Sherlock scans your store automatically every day during 1:00 AM - 6:00 AM in your store's local timezone.

We chose these hours because:

  • It's typically your store's quietest time
  • Scanning won't affect your customers' experience
  • Any issues are caught before your busy daytime hours

You can also run an on-demand scan anytime from your dashboard (up to 5 per day).

After each scan, Sherlock calculates an overall risk level:

Level What It Means
Low Few or no concerning changes. Your store looks healthy.
Medium Some changes detected that might need attention.
High Significant changes detected. We recommend investigating.

Factors that increase risk: many files changed, new scripts added, CSS conflicts found, slow page load, or poor performance scores.

You can run up to 5 on-demand scans per day. This limit resets at midnight.

Daily automatic scans don't count toward this limit ‚Äî those happen automatically regardless.

Most merchants only need 1-2 on-demand scans after installing or removing an app.

App Conflicts & Issues

A CSS conflict happens when an app adds styling code that accidentally affects other parts of your store.

The problem: Some apps add CSS with generic names like .button or .card that change every button or card on your store ‚Äî not just the app's own elements.

Symptoms you might see:

  • Buttons look different after installing an app
  • Text colors or fonts changed unexpectedly
  • Layout shifts or spacing issues
  • Elements overlapping or hidden

Sherlock detects these risky CSS patterns and tells you which app is responsible.

Orphan code is leftover code from apps you've uninstalled. Many apps don't clean up after themselves when you remove them.

Why it's a problem:

  • Slows down your store (loading code that does nothing)
  • Can cause unexpected behavior or errors
  • May conflict with new apps you install
  • Clutters your theme files

Sherlock detects orphan code by comparing scripts in your theme against your currently installed apps. If we find code from an app you no longer have, we'll flag it.

Sherlock maintains a database of apps that are known to conflict with each other. This information comes from:

  • Shopify Community Forums
  • Reddit r/shopify discussions
  • Facebook Shopify groups
  • Common support patterns

Examples of conflict pairs:

  • Page Builders: PageFly + GemPages (both heavily modify theme code)
  • Review Apps: Loox + Judge.me (duplicate widgets, layout issues)
  • Subscription Apps: ReCharge + Bold (can break checkout)

If you have two conflicting apps installed, Sherlock will warn you.
